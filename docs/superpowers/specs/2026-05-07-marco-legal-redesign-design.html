Marco Legal — Rediseño UI (Fase A)

Rediseño completo de /marco-legal para consumir el contenido legal rico que ya existe.

Spec aprobado Branch: feature/product-refinement Pendiente plan de implementación

Contexto

La vista actual muestra 5 categorías con contenido placeholder pobre. Al mismo tiempo existe src/data/legal-references.ts con 391 líneas de contenido legal rico por permiso (fuentes con artículos, consecuencias, proceso, documentos, costos, links) que no se consume. Las tablas DB legal_* existen pero están vacías.

Problema principal El usuario encuentra la vista inservible ("está como una puta mierda"). Queremos que la UI muestre el contenido rico que ya tenemos.

Scope

✅ Incluido (Fase A)

  • Rediseño de /marco-legal y rutas hijas
  • Consumo de legal-references.ts
  • Mapping PermitType → Category
  • UI responsiva desktop + mobile

🔒 Excluido (Fase B)

  • Validación legal del contenido
  • Seed de tablas DB legal_*
  • Permisos nuevos (ambiental, laboral)
  • Print / export PDF / UI de edición

Data layer

Fuente única: src/data/legal-references.ts. Se agrega mapping de categorías:

export type LegalCategory =
  | 'funcionamiento' | 'sanitario' | 'ambiental' | 'laboral' | 'seguridad';

export const PERMIT_TO_CATEGORY: Record<PermitType, LegalCategory> = {
  ruc: 'funcionamiento',
  patente_municipal: 'funcionamiento',
  uso_suelo: 'funcionamiento',
  rotulacion: 'funcionamiento',
  bomberos: 'seguridad',
  arcsa: 'sanitario',
};

Selectores in-memory O(n) con n=6. No hay paginación ni cache.

Accesibilidad

  • Tabs / segmented: role="tablist", role="tab", aria-selected, navegación con flechas
  • Chips: <button> con aria-pressed
  • Breadcrumb: <nav aria-label="Breadcrumb">
  • Search input: aria-label obligatorio
  • Disclaimer: role="note"
  • Links externos: rel="noopener noreferrer", target="_blank", icono visible
