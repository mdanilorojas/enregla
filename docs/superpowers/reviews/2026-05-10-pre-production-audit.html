Acciones del operador antes de vender

Tres acciones requieren intervención manual (no son fixes de código):

  1. Setear CRON_SECRET en dos lugares con el mismo valor: 
    -- En Supabase SQL editor:
ALTER DATABASE postgres SET app.cron_secret TO '<64-char random>';

-- En terminal:
supabase secrets set CRON_SECRET=<mismo valor>
    Hasta que ambos estén, el cron responde 401 y no se envían emails.
  2. Activar "Leaked Password Protection" en Supabase Dashboard → Auth → Password Strength. Es un toggle en dashboard, no hay DDL para automatizarlo.
  3. Mover el from: de Resend al dominio verificado noreply@enregla.ec (actual default: onboarding@resend.dev). Setear RESEND_FROM como secret en la edge function y verificar DKIM/SPF del dominio.

Los 20 BLOCKERs — cada uno verificado en estado LIVE

Un agente verificador dedicado corrió consultas read-only contra Supabase prod tras cada fix. No confía en el archivo de migración: confía en pg_policies, storage.buckets, pg_proc. Cada fila = PASS confirmado.

BLOCKER B1 · Cron dead for 10 days (extensions.http_post does not exist)

Fix: Reemplazado con net.http_post (pg_net) + header x-cron-secret desde app.cron_secret DB setting.

Verificado: cron.job.command llama net.http_post. Activo · schedule 0 8 * * *.

BLOCKER B2 · Edge function publicly invokable (no auth, CORS *)

Fix: Se requiere header x-cron-secret validado contra CRON_SECRET env. CORS restringido a ALLOWED_ORIGIN (default https://app.enregla.se). Método no-POST rechazado.

BLOCKER B3 · Service-role JWT habría quedado plaintext en cron command

Fix: El secreto se lee vía current_setting('app.cron_secret'). El service_role key nunca sale del env de la edge function.

BLOCKER B4 · No idempotency — rerun del cron duplicaría emails

Fix: Índice UNIQUE uq_notification_logs_user_permit_type_day sobre (user_id, permit_id, notification_type, sent_at::date). La edge function pre-filtra los ya-enviados del día.

BLOCKER B5 · get_expiring_permits() callable por anon/authenticated

Fix: REVOKE EXECUTE from anon/authenticated/PUBLIC. Grant a service_role únicamente.

BLOCKER B6 · Storage bucket permit-documents era public=true

Fix: public=false · file_size_limit=5242880 (5MB) · allowed_mime_types=[pdf,png,jpeg].

BLOCKER B7 · Storage RLS OR-short-circuit → cualquier usuario R/W/D cualquier archivo

Fix: 4 policies reescritas con AND-semantics. authenticated solo para user-scoped; anon solo vía public-link con token + expires_at.

BLOCKER B8 · Anon 50MB upload DoS

Fix: INSERT role restringido a authenticated. Size 5MB. UUID regex sobre permit folder.

BLOCKER B9 · URLs del inspector vivían para siempre (sin revocación posible)

Fix: Todos los getPublicUrl migrados a createSignedUrl(300s). Storage policy pública exige pl.is_active AND expires_at IS NULL OR expires_at > now().

BLOCKER B10 · documents.file_path podía apuntar a archivo de otra empresa

Fix: CHECK constraint file_path LIKE 'permits/' || permit_id::text || '/%'.

BLOCKER B11 · partners leak a todo usuario autenticado (CRM interno)

Fix: Nueva columna profiles.is_staff + 4 policies staff-only (partners_staff_{select,insert,update,delete}).

BLOCKER B12 · leads leak (PII: nombres, emails, teléfonos, status CRM)

Fix: Mismo patrón staff-only en leads_staff_{select,update,delete}. Anon INSERT conservado (captura desde landing) con CHECK constraints de largo.

BLOCKER B13 · companies_insert WITH CHECK (true)

Fix: WITH CHECK (auth.uid() IS NOT NULL AND EXISTS (profiles WHERE company_id IS NULL)).

BLOCKER B14 · Triggers de onboarding NO instalados → loop infinito para usuarios reales

Fix: Recreados trigger_auto_create_permits (locations) y companies_auto_assign_to_profile (companies).

BLOCKER B15 · Policy de profiles con UUID demo hardcoded

Fix: Reemplazado por visibilidad por teammate + DEMO_COMPANY_ID. Único UUID hardcoded restante = la constante central de la política.

BLOCKER B16 · get_public_permits sin check de expiración · mutaciones anon

Fix: Separada la lectura (get_public_permits, ahora con gate expires_at > now()) de la mutación (increment_public_link_view). Grants tightened.

BLOCKER B17 · Cero ErrorBoundary

Fix: <ErrorBoundary> envolviendo <App/> en main.tsx. Fallback UI amigable + stack trace solo en DEV.

BLOCKER B18 · QueryClientProvider nunca montado

Fix: Montado en main.tsx. Cache .clear() al SIGNED_OUT y dentro de useAuth.signOut.

BLOCKER B19 · Sin CSP — XSS expondría tokens de localStorage

Fix: CSP estricta + HSTS en vercel.json: default-src 'self' · frame-ancestors 'none' · script-src 'self' · connect-src restringido a supabase.co.

BLOCKER B20 · OAuth implicit flow vulnerable a hash-token fixation

Fix: Cliente Supabase con flowType: 'pkce'.
